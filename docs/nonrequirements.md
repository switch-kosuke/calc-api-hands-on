# 非機能要件仕様書

## 1. インフラストラクチャ要件

### 1.1 クラウドプロバイダー
- **プロバイダー**: Microsoft Azure
- **リージョン**: 東日本 (Japan East)
- **理由**: 日本国内からの低レイテンシーアクセスを実現

### 1.2 コンピューティングサービス
- **サービス**: Azure Functions
- **プラン**: Consumption Plan (従量課金プラン)
- **特徴**:
  - サーバーレスアーキテクチャ
  - リクエストベースの自動スケーリング
  - 使用した分のみ課金

### 1.3 リソース構成
```
Azure Region: Japan East
├── Function App (Consumption Plan)
    ├── /multiply (HTTP Trigger)
    └── /divide (HTTP Trigger)
```

## 2. 開発環境要件

### 2.1 プログラミング言語
- **言語**: Python
- **バージョン**: 3.11

## 3. 性能要件

### 3.1 レスポンスタイム
- **目標値**: 5秒以内
- **測定条件**: リクエスト送信からレスポンス受信まで
- **対象**: 正常系のすべてのリクエスト

### 3.2 同時実行数
- **想定値**: 1リクエスト
- **説明**: 同時に処理されるリクエスト数は1つを想定
- **スケーリング**: Consumption Plan により自動的に処理されるが、現時点では同時実行を考慮しない

### 3.3 スループット
- **制約**: Azure Functions Consumption Plan のデフォルト制限に従う
- **特別な要求**: なし

### 3.4 Consumption Plan の制限
- **メモリ**: 設定不要(デフォルト)
- **実行時間**: 設定不要(デフォルト: 5分)
- **考慮事項**: 特別な設定は不要

## 11. 制約事項

### 11.1 Azure Functions Consumption Plan の制限
- **実行時間**: 最大5分(デフォルト設定)
- **メモリ**: 1.5 GB(デフォルト)
- **同時実行数**: リージョンごとに制限あり(通常200-300)
- **コールドスタート**: 初回実行時に遅延の可能性

### 11.2 コスト制約
- **予算**: 特に規定なし
- **課金モデル**: 実行回数と実行時間による従量課金

### 11.3 技術的制約
- **Python バージョン**: 3.11 固定
- **Azure Functions**: v4 ランタイム(Python 3.11対応)
